Component Standards • Modals

Responsive Confirmation Dialogs

Standardized modal interfaces for critical user decisions, destructive actions (such as account or repository deletion), status warnings, and session management. Equipped with backdrop blur, keyboard accessibility (Esc to close), and focus trapping.

delete_forever Destructive Action #modal-delete

Account / Data Deletion

Used when an action will permanently destroy data or user records. Features a distinct red warning badge, high-contrast action buttons, and clear explanation of irreversibility.

Typical Use Cases:

  • Deleting a user account or organization
  • Removing a production database or repository
  • Revoking permanent security licenses
warning Status Warning #modal-warning

Revoke Access / Downgrade

Used for high-impact changes that alter active services, billing plans, or security keys without destroying stored data. Prompts the user to verify intent before execution.

Typical Use Cases:

  • Revoking API access tokens or webhook keys
  • Downgrading subscription tier or removing seats
  • Disabling multi-factor authentication (MFA)
rocket_launch Information • Action #modal-publish

Production Publish / Deploy

Used when initiating significant system broadcasts, global notifications, or live production deployments. Provides a final review checkpoint before triggering automated workflows.

Typical Use Cases:

  • Deploying software releases to live production
  • Sending mass email announcements to subscribers
  • Applying global configuration overrides
logout Session Management #modal-logout

Session Logout Confirmation

Ensures users do not accidentally terminate active working sessions, especially when uncommitted forms or drafts might be open in background tabs.

Typical Use Cases:

  • Logging out from dashboard navigation menus
  • Terminating remote authenticated sessions
  • Switching active organization profiles
